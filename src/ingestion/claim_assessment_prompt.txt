Given a claim and its source paragraph, evaluate the claim across 7 quality dimensions plus an overall confidence rating. Output a JSON object with exactly these 8 fields. Use the exact string values listed for each field.

DIMENSIONS TO ASSESS:

1. specificity — Does the claim name a specific target, metric, program, or quantity?
   - "high": claim names a specific metric AND quantified value OR a named program with specific scope
   - "medium": claim names a specific metric or named program but lacks quantification or precise scope
   - "low": vague aspiration with no specific metric, value, or named program

2. time_boundedness — Does the claim have a deadline or specific timeframe?
   - "bounded": specific year, date, or timeframe stated (e.g., "by 2030", "since 2015")
   - "ongoing": continuous program or commitment with no defined endpoint
   - "unbounded": no temporal reference of any kind

3. measurability — Can the claim's content be measured?
   - "quantified": contains a specific number with unit (%, tonnes, count, ratio)
   - "qualitative_measurable": no number but the outcome could in principle be verified (e.g., "implemented program X")
   - "not_measurable": purely abstract or aspirational, no measurable outcome

4. verifiability — Can a third party verify this claim against evidence?
   - "externally_verifiable": claim references external standards (SBTi, GRI, TCFD, ISO, NRDC, etc.) or third-party partners that provide independent verification
   - "internally_verifiable": claim is checkable but only against the company's own internal data
   - "not_verifiable": no specific facts that could be checked against any data source

5. status — Is the claim about a past event, ongoing program, or future target?
   - "past_achievement": describes something already completed or achieved
   - "ongoing_program": describes an active, continuous program or practice
   - "future_target": describes a goal to be achieved in the future

6. scope_clarity — Does the claim clearly define what is covered (geographic, operational, segment scope)?
   - "clear": specific geographic regions, operational segments, or scope categories named (Scope 1/2/3, named regions, named business units)
   - "partial": some scope information provided but incomplete (e.g., region named but operations not specified)
   - "vague": no scope information, or only vague references ("our operations", "globally")

7. conditionality — Is the claim conditional, unconditional, or aspirational?
   - "unconditional": firm commitment without dependencies on external factors (hedging language like "aim to" is still unconditional unless explicit conditions are stated)
   - "conditional": explicit conditions stated ("subject to regulatory frameworks", "provided that technology becomes available", "where commercially feasible")
   - "aspirational": values or beliefs stated rather than actions or commitments ("we believe in", "we are committed to a sustainable future")

8. assessment_confidence — How confident are you in this overall assessment?
   - "high": the claim and paragraph provide enough context for a clear assessment across all dimensions
   - "medium": some dimensions are ambiguous or borderline
   - "low": the claim is unclear or the paragraph context conflicts with the claim text

CRITICAL RULES:
- Output ONLY a valid JSON object with exactly these 8 fields. No explanations, no markdown.
- For each field, use ONLY the exact string values listed above. Do not invent new values.
- Use the paragraph context to resolve ambiguity (e.g., if the claim text alone is vague but the paragraph provides scope or timeframe, use that information).
- Hedging language ("aim to", "plan to", "strive to") is NOT conditionality unless explicit external conditions are stated.
- A claim can be "quantified" in measurability even if the quantified value is in the paragraph rather than the claim text itself.
